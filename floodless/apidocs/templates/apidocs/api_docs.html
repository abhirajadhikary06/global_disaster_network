API Docs & Dataset

Download Dataset Dataset Link

Base URL

https://floodless.example.com/api/disasters/

Endpoints

GET /disasters/

This endpoint retrieves disaster records and supports filtering by parameters.


Query Parameters

Parameter Type Description
year Integer Filter by year (e.g., 2020)
disaster_type String Type of disaster (e.g., Flood, Earthquake)
country String Filter by country
region String Filter by region
location String Filter by specific location
limit Integer Max results per page (default: 100)

Example API Request

GET https://abhirajadhikary06.github.io/DisasterAPI/disasters.json?year=2020&disaster_type=Flood&limit=5

Example API Response

{
"count": 1,
"next": null,
"previous": null,
"results": [
    {
        "year": 2020,
        "disaster_type": "Flood",
        "country": "India",
        "region": "Kerala",
        "location": "Alappuzha",
    }
]
}

Authentication

The API requires an API key. Include your key in the request header:

Authorization: Bearer YOUR_API_KEY

To obtain an API key, contact admin@floodless.example.com.


Error Codes

Code Description
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid or missing API key
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error

Rate Limiting

To ensure fair usage, the API enforces rate limits:

  • Unauthenticated users: 100 requests per hour
  • Authenticated users: 1,000 requests per hour

Exceeding the limit will result in a 429 Too Many Requests response.


Pagination

Results are paginated with a default page size of 100 records. Use the page query parameter to navigate through pages. Metadata about pagination is included in the response:

  • count: Total number of records
  • next: URL for the next page
  • previous: URL for the previous page

Sorting

Results can be sorted by specific fields using the ordering query parameter:

  • ordering=-year: Sort by year in descending order
  • ordering=disaster_type: Sort by disaster type in ascending order

Use Cases

This API can be used for various purposes, including:

  • Data visualization on interactive maps or dashboards
  • Research and analysis of historical disaster data
  • Building alert systems for early warnings
  • Developing mobile apps to provide disaster-related information
